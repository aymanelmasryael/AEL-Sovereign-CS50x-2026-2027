1. التهيئة وضبط الهوية السيادية (Setup & Configuration)

الأوامر الأساسية لتعريف هويتك كمهندس نظم وتهيئة المستودعات المحلية واستنساخ المشاريع.

zsh - config_setup
# 1. ضبط اسم المالك السيادي عالمياً في جهازك $ git config --global user.name "Ayman Elmasry" # 2. ضبط البريد الإلكتروني الرسمي للمالك $ git config --global user.email "info@aymanelmasry.com" # 3. فحص إعدادات الهوية الحالية للتأكد من صحتها $ git config --list # 4. تهيئة مستودع محلي جديد داخل أي مجلد $ git init # 5. استنساخ مستودع سحابي قائم إلى جهازك المحلي $ git clone https://github.com/aymanelmasryael/AEL-Sovereign-CS50x-2026-2027.git

2. الفحص، المراقبة، والمقارنة الذرية (Inspection & Auditing)

أوامر الرادار البرمجي لمعرفة حالة الكود وتتبع التعديلات وسجلات الحفظ بدقة متناهية.

zsh - inspection
# 1. فحص حالة الملفات ومعرفة ما تم تعديله أو إضافته $ git status # 2. عرض الفروقات البرمجية الدقيقة (السطور التي تم تعديلها قبل حفظها) $ git diff # 3. عرض سجل الحفظ (Commits) بأسلوب معماري ناصع ومختصر $ git log --oneline --graph --all # 4. عرض تفاصيل آخر عملية حفظ تمت في المستودع $ git show HEAD

3. التجهيز والتوثيق السيادي (Staging & Committing)

أوامر تجهيز الملفات وختمها بالسجلات البيروقراطية المنظمة وتعديل وثائق الحفظ.

zsh - commit
# 1. إضافة كافة الملفات والتعديلات إلى منصة التجهيز (Staging) $ git add . # 2. إضافة ملفات محددة بدقة $ git add README.md AEL-Sovereign-CS50x-2026-2027.png # 3. إنشاء وثيقة حفظ (Commit) مع عنوان معماري احترافي $ git commit -m "🚀 [v1.0] Sovereign System Architect Initialization" # 4. تعديل آخر وثيقة حفظ (في حال أردت تغيير الوصف أو إضافة ملف نسيته) $ git commit --amend -m "🚀 [v1.0] Sovereign System Architect Initialization (Updated)"

4. إدارة الفروع وسير عمل البيئات المحمية (Branching & Workflows)

الأسلوب المتبع للتعامل مع الفروع الجانبية والبيئات ذات قواعد الحماية الصارمة (Branch Protection).

zsh - branching
# 1. عرض قائمة الفروع المحلية المتاحة في جهازك $ git branch # 2. إعادة تسمية الفرع الحالي ليكون بالاسم القياسي المعتمد (main) $ git branch -M main # 3. إنشاء فرع جانبي جديد والانتقال إليه فوراً (الأسلوب المفضل لطلبات السحب) $ git checkout -b feature-sovereign-upgrade # 4. الانتقال السلس بين الفروع المتاحة $ git switch main # 5. حذف فرع محلي بعد الانتهاء منه $ git branch -d feature-sovereign-upgrade

5. التزامن السحابي والمحاذاة الصارمة (Remote Synchronization)

أوامر التخاطب المباشر مع الخوادم لرفع أو سحب الملفات ومحاذاة الفروع السحابية.

zsh - remote_sync
# 1. ربط مستودعك المحلي بالمستودع السحابي الرسمي $ git remote add origin https://github.com/aymanelmasryael/AEL-Sovereign-CS50x-2026-2027.git # 2. فحص روابط الاتصال السحابي الحالية $ git remote -v # 3. سحب آخر التحديثات من الخادم دون دمجها (للفحص الأمني) $ git fetch origin # 4. سحب التحديثات السحابية ودمجها بالفرع المحلي مباشرة $ git pull origin main # 5. دفع الكود المحلي إلى السحاب مع تثبيت مسار الاتصال $ git push -u origin main # 6. دفع فرع جانبي جديد إلى السحاب $ git push -u origin feature-sovereign-upgrade # 7. حذف فرع جانبي من السحاب مباشرة $ git push origin --delete feature-sovereign-upgrade

6. طوارئ النظام واستعادة النقاء التام (Undoing & Emergency Resets)

أوامر الصيانة الطارئة للتراجع عن الأخطاء وإعادة ضبط المستودع ليكون ناصع النظافة ومطابقاً للسحاب 100%.

zsh - emergency_reset
# 1. إلغاء تعديلات ملف لم يتم حفظه وإعادته لحالته السابقة $ git restore server_check50_execution.py # 2. إزالة ملف من منصة التجهيز (Staging) مع الحفاظ على التعديلات بداخله $ git reset ael_supabase_connector.js # 3. حذف آخر عملية حفظ (Commit) محلياً والتخلص من تعديلاتها نهائياً $ git reset --hard HEAD~1 # 4. الأمر الصارم لمطابقة الكود المحلي بالسحاب 100% وإزالة أي أخطاء أو شوائب محلية $ git reset --hard origin/main

7. أوامر التحكم السحابي الخارق عبر `gh CLI`

الأسلحة المتقدمة التي تميز كبار مهندسي النظم، وتتيح التحكم بسيرفرات GitHub وطلبات السحب والأتمتة من الطرفية مباشرة.

zsh - gh_cli_elite
# 1. المصادقة وتسجيل الدخول السحابي الآمن إلى حسابك في GitHub $ gh auth login # 2. إنشاء مستودع سحابي جديد ورفع الكود إليه بضغطة زر واحدة $ gh repo create aymanelmasryael/AEL-New-Project --public --source=. --remote=origin --push # 3. السطر الذهبي لضبط إعدادات المستودع (تفعيل النقاشات وفرض الدمج النظيف والحذف التلقائي) $ gh repo edit aymanelmasryael/AEL-Sovereign-CS50x-2026-2027 --delete-branch-on-merge --enable-discussions --enable-squash-merge # 4. فتح طلب سحب رسمي (Pull Request) من موجه الأوامر $ gh pr create --title "🎨 [UI/UX] Showcasing Master Architectural Banner" --body "Injecting master banner for canonical visual authority." # 5. عرض قائمة طلبات السحب المفتوحة حالياً في المستودع $ gh pr list # 6. فحص تفاصيل وحالة طلب سحب محدد برقم الـ ID $ gh pr view 2 # 7. السطر الصارم لدمج طلب السحب أوتوماتيكياً بصلاحيات المالك العليا وحذف الفرع الجانبي $ gh pr merge 2 --squash --delete-branch --admin # 8. استعراض قائمة أحدث عمليات الفحص والبناء الجارية في سيرفرات GitHub Actions $ gh run list --limit 5 # 9. سحب تقرير الحالة الحية لعملية فحص محددة ومعرفة تفاصيل النجاح أو الفشل $ gh run view 28381788686
