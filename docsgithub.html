ميثاق التخاطب السيادي مع GitHub

دليل هندسي فاخر وموثق لكافة الأوامر السحابية والمحلية الخاصة بإدارة مستودعات AEL السيادية بأسلوب كبار مهندسي النظم.

1. التهيئة والإضافة المحلية (Local Initialization)

الأوامر المستخدمة لتهيئة القلعة المحلية داخل مجلد المشروع وتجهيز الملفات الجديدة للرفع.

zsh - local_init
# 1. تهيئة المستودع المحلي داخل مجلد مشروعك $ git init # 2. فحص حالة الملفات ومعرفة ما تم تعديله $ git status # 3. إضافة كافة الملفات والأنظمة إلى منصة التجهيز $ git add . # (أو إضافة ملفات محددة بدقة ذرية) $ git add server_check50_execution.py ael_supabase_connector.js

2. التوثيق وختم الملكية (Enterprise Commits)

اعتماد العناوين المعمارية الصارمة والرموز التعبيرية (Emojis) لتوثيق هوية الإصدارات بدقة.

zsh - commit
# إنشاء وثيقة الحفظ (Commit) مع وصف احترافي للنسخة $ git commit -m "🚀 [v4.0] Full-Stack Enterprise Upgrade: Adding Isolated Docker Execution Engine & Supabase Cloud Persistence Connector"

3. الربط السحابي والرفع المباشر (Direct Push)

إنشاء جسر الاتصال الأولي بالمستودع السحابي الرسمي على GitHub ودفع الكود قبل تفعيل درع الحماية.

zsh - remote_push
# 1. ربط مستودعك المحلي بالمستودع السحابي الرسمي على GitHub $ git remote add origin https://github.com/aymanelmasryael/AEL-Sovereign-CS50x-2026-2027.git # 2. إعادة تسمية الفرع الرئيسي ليكون بالاسم المعتمد عالمياً $ git branch -M main # 3. دفع الكود إلى السحاب مع حفظ مسار الاتصال الدائم $ git push -u origin main

4. الضبط المعماري المتقدم عبر `gh CLI`

السطر الذهبي الخارق لضبط إعدادات المستودع السحابي، تفعيل النقاشات (Discussions)، وفرض أسلوب الدمج النظيف (Squash Merge).

zsh - gh_repo_edit
# ضبط إعدادات المستودع السحابي مباشرة من الطرفية $ gh repo edit aymanelmasryael/AEL-Sovereign-CS50x-2026-2027 --delete-branch-on-merge --enable-discussions --enable-squash-merge --enable-merge-commit=false --enable-rebase-merge=false

5. سير عمل النخبة (Zero-Trust PR Workflow)

الأسلوب المتبع بعد تفعيل درع حماية الفرع الرئيسي (Branch Protection)، لرفع صورة الغلاف والتعديلات عبر الفروع الجانبية وطلبات السحب.

zsh - pull_request
# 1. إنشاء فرع جانبي جديد والانتقال إليه فوراً $ git checkout -b add-hero-banner # 2. رفع الفرع الجانبي إلى خوادم GitHub $ git push -u origin add-hero-banner # 3. فتح طلب سحب رسمي (Pull Request) من داخل موجه الأوامر مباشرة $ gh pr create --title "🎨 [UI/UX] Showcasing AEL Sovereign Cover Banner in Repository Root & README" --body "Adding the magnificent hero cover image banner to establish canonical visual authority."

6. المراقبة والاتصال الحي بسيرفرات `GitHub Actions`

أوامر الرادار السحابي لمراقبة خطوط الأنابيب (Pipelines) وبناء صور Docker ونشر الموقع في الوقت الفعلي.

zsh - gh_run
# 1. استعراض قائمة أحدث عمليات البناء والفحص الجارية في السحاب $ gh run list --limit 3 # 2. سحب تقرير الحالة الحية لعملية فحص محددة ومعرفة تفاصيلها $ gh run view 28381788686
