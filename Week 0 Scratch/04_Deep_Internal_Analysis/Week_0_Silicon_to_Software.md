## 📦 الوحدة 4: Binary & Sub-Silicon Physics (فيزياء السيليكون)

```text
Transistor ──> Logic Gates ──> Full Adder ──> ALU ──> CPU Registers ──> Machine Code (0/1)
```

### 💡 لماذا يتحدث الحاسوب بالنظام الثنائي (Binary - Base 2) فقط؟
لماذا لا يعتمد الحاسوب النظام العشري (Base 10) المألوف للبشر؟
*   **التبرير الفيزيائي الهندسي:** يعمل الحاسوب بالتيار الكهربائي. محاولة تصميم أجهزة قياس دقيقة تقرأ 10 مستويات جهد مختلفة (مثلاً من 0 إلى 9 فولت) بسرعة مليارات المرات في الثانية ستؤدي حتماً إلى أخطاء فادحة بسبب الضوضاء الكهربائية (Electrical Noise) والتداخل الكهرومغناطيسي.
*   **الحل الثنائي الأنيق (On / Off):** تبسيط المنظومة إلى حالتين فقط. إما يوجد جهد كهربائي (جهد مرتفع = `1`) أو لا يوجد (جهد منخفض = `0`).

### 🚀 رحلة الصعود التجريدي (Abstraction Ascent)
1. **الترانزستور (Transistor):** مفتاح كهروميكانيكي دقيق جداً (بضع نانومترات) مبني من السيليكون وأشباه الموصلات (`CMOS`). يعمل كمفتاح: يمرر الكهرباء (`1`) أو يحجبها (`0`).
2. **البوابات المنطقية (Logic Gates):** بدمج ترانزستورين أو أكثر، نبني بوابات مثل `AND`, `OR`, `NOT`, و `NAND`. (بوابة `NAND` تعتبر بوابة عالمية `Universal Gate` يمكن بناء معالج كامل منها).
3. **الجامع الكامل (Full Adder):** بدمج بوابات `XOR` و `AND`، نصل إلى دائرة تجمع خانتين ثنائيتين (Bits) وتنتج المجموع (Sum) والمحمول (Carry).
4. **وحدة الحساب والمنطق (ALU):** بدمج ملايين الـ Adders، نحصل على الـ `Arithmetic Logic Unit` داخل الـ CPU.
5. **المعالج ومسجلات الذاكرة (CPU & Registers):** تُخزن النتيجة في مسجلات سريعة جداً (`Registers` مثل `RAX`, `RBX`).
6. **لغة الآلة (Machine Code):** التعليمات البرمجية التي تتحكم في هذه الدوائر تُكتب كنبضات من الـ `0` و `1` (مثلاً `10110000 01100001` تعني وضع القيمة 97 في المسجل `AL`).

## 📦 الوحدة 5: ASCII Table Internal Mechanics (عبقرية جدول ASCII)

### 💡 التشريح الرياضي والداخلي
في جدول `ASCII`، يُخصص بايت واحد (8 بتات) لتمثيل الرموز.
*   الحرف `A` الكبير = `65` (بالنظام الثنائي: `01000001`)
*   الحرف `B` الكبير = `66` (بالنظام الثنائي: `01000010`)
*   الحرف `a` الصغير = `97` (بالنظام الثنائي: `01100001`)

### ⚙️ لماذا الفارق بين الحرف الكبير والصغير هو بالضبط 32؟ (The Genius of 32)
لم يكن اختيار الرقم 65 لـ `A` و 97 لـ `a` عشوائياً، بل هو عبقرية هندسية خالصة لتسهيل عمل المعالج (CPU).
الفارق بين الحرف الكبير والصغير هو `97 - 65 = 32`. 
الرقم `32` في النظام الثنائي يمثل الخانة السادسة: `00100000`.

```text
Character 'A':  0 1 0 0 0 0 0 1  (65)
Number 32    :  0 0 1 0 0 0 0 0  (32)  <-- (Bitmask)
----------------------------------- (Bitwise OR)
Character 'a':  0 1 1 0 0 0 0 1  (97)
```

### 🚀 ماذا يحدث داخل المعالج (CPU Case Flipping)؟
عندما تطلب من برنامج مثل `Word` أو وظيفة `tolower()` في لغة C تحويل حرف من كبير إلى صغير، فإن المعالج لا يبحث في جداول معقدة. المعالج يقوم بعملية معالجة بتات سريعة جداً (Bitwise Operation) في دورة معالج واحدة (Single Clock Cycle):
*   **للتحويل من كبير إلى صغير (Lowercasing):** يقوم المعالج بإجراء `Bitwise OR` مع `00100000` (تشغيل بت الـ 32).
*   **للتحويل من صغير إلى كبير (Uppercasing):** يقوم المعالج بإجراء `Bitwise AND` مع `11011111` (إطفاء بت الـ 32).
*   **لعكس الحالة (Case Inversion):** يقوم المعالج بإجراء `Bitwise XOR` مع `00100000`.

هذا التصميم يوفر ملايين الدورات الذاكرية (Memory Cycles) ويجعل معالجة النصوص فائقة السرعة في محركات مثل `V8` و `SQLite`.

## 📦 الوحدة 9: Pseudo Code to Silicon (من الشفرة الزائفة إلى السيليكون)

```text
[ Pseudo Code ] ──> [ C / Python Code ] ──> [ AST Parser ] ──> [ Compiler / IR ] ──> [ Assembly ] ──> [ Machine Code ]
```

### 🔬 رحلة التحول إلى السيليكون
كيف تتحول عبارة مثل `If page matches, call Mike` إلى حركة إلكترونات؟
1. **الشيفرة المصدرية (Source Code):** يترجم المبرمج الـ `Pseudo Code` إلى لغة مثل `C` أو `Python`.
2. **المحلل القواعدي (Lexer & Parser):** يقوم المترجم (Compiler مثل `GCC` أو `Clang`) بتحويل النصوص إلى شجرة نحو مجردة **`AST (Abstract Syntax Tree)`**.
3. **التمثيل الوسيط (Intermediate Representation - IR):** يتم تحويل الـ `AST` إلى شيفرة وسيطة لعمل التحسينات (Optimizations) مثل حذف الأكواد الميتة.
4. **لغة التجميع (Assembly):** يُترجم الـ `IR` إلى تعليمات خاصة بمعمارية المعالج (مثلاً `x86-64` أو `ARM64`) مثل `CMP` (مقارنة) و `JMP` (قفز).
5. **لغة الآلة والتنفيذ (Machine Code):** يحول الـ `Assembler` الـ Assembly إلى ملف تنفيذي مليء بـ `0` و `1` (`ELF` في Linux أو `Mach-O` في macOS). يقرأ الـ CPU هذه البايتات، ويفتح بوابات السيليكون لتنفيذ الأوامر.
