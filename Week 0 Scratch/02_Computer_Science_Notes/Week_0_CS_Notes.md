## 📦 الوحدة 1: What is Computer Science?

### 💡 المفهوم الجوهري
علم الحاسب (Computer Science) ليس علم البرمجة، وليس علم صيانة الأجهزة. 
إنه **علم دراسة المعلومات (Information Study)**:
1. **التمثيل (Representation):** كيف نترجم الأفكار والأرقام والنصوص والصور إلى تنسيق رقمي قياسي (Standardized Data).
2. **المعالجة (Processing):** كيف نتلاعب بتلك المعلومات وننقلها من حالة إلى أخرى بأسرع وأكفأ الطرق الممكنة.

## 📦 الوحدة 2: Computational Thinking (التفكير الحسابي)

### 💡 تعريفه وأركانه
التفكير الحسابي هو صياغة المشكلات وحلولها بطريقة تمكن الحاسوب (أو الإنسان) من تنفيذها كخطوات منطقية غير قابلة للتأويل. 

#### الأركان الأربعة للتفكير الحسابي:
1. **التفكيك (Decomposition):** تقسيم المشكلات المعمارية المعقدة إلى مشكلات جزئية يسهل إدارتها.
2. **تمييز الأنماط (Pattern Recognition):** البحث عن القواعد المتكررة لإعادة استخدام الحلول الكفؤة.
3. **التجريد (Abstraction):** إخفاء التعقيدات التحتية (مثل كيفية تحريك الإلكترونات في السيليكون) والتركيز على الواجهة الأساسية (مثل الدوال البرمجية).
4. **تصميم الخوارزميات (Algorithm Design):** بناء خطوات متسلسلة حتمية (Deterministic) تقود حتماً إلى المخرج المطلوب.

### ⚙️ الفوارق المعمارية بين العقل البشري والآلة
*   **العقل البشري (Intuitive & Massively Parallel):** يمتلك القدرة على تمييز الأنماط تناظرياً وبالتوازي الهائل. يمكن للإنسان فتح دليل هاتف ضخم والوصول إلى حرف معين بالحدس والتقريب.
*   **الحاسوب (Deterministic & Sequential):** الآلة كائن متسلسل يعمل وفق دورات زمنية دقيقة (Clock Cycles). لا تملك الآلة حدساً؛ تحتاج إلى عنوان ذاكري محدد بدقة (Exact Memory Address) وتعليمات خطوة بخطوة.

## 📦 الوحدة 3: Input → Processing → Output (The Black Box Model)

```text
+-------------------+       +-----------------------+       +--------------------+
|   Input (Data)    |  ──>  | Processing (Black Box)|  ──>  |   Output (Goal)    |
| (Keyboard, Files) |       |  (CPU, Algorithms)    |       | (Screen, Disk, AI) |
+-------------------+       +-----------------------+       +--------------------+
```

### 💡 تحليل الصندوق الأسود (Black Box)
كل نظام في علوم الحاسب، سواء كان وظيفة برمجية صغيرة أو محرك بحث عملاق، يخضع لنموذج الصندوق الأسود:
*   **Input:** المشكلة المطلوب حلها أو البيانات الخام الممثلة بنظام متفق عليه (أصفار ووحدات).
*   **Processing:** "الخلطة السرية" (Secret Sauce) أو الخوارزمية التي يقوم المعالج بتنفيذها لحل المشكلة.
*   **Output:** النتيجة والهدف النهائي.

## 📦 الوحدة 11: The Abstraction Ladder (سُلّم التجريد: من التكرار إلى المعاملات)

استعرض ديفيد مالان رحلة رائعة لكيفية تطور الفكر البرمجي للتعامل مع مشكلة تكرار الأكواد. في منهجية AEL، نطلق على هذه الرحلة **سُلّم التجريد (The Abstraction Ladder)**، وهو يعكس كيف ترتقي جودة المعمارية البرمجية خطوة بخطوة.

```text
[ Stage 1: Repetitive Blocks ] ──> [ Stage 2: Control Flow Loops ] ──> [ Stage 3: Custom Function ] ──> [ Stage 4: Parameterized Function ]
```

### 🔬 درجات السُلّم التجريدي:
1. **الدرجة 1: الأكواد المكررة (Repetitive Boilerplate Code):**
   *   *التنفيذ:* وضع 3 كتل `play sound [Meow]` متتالية، بينها `wait 1 seconds`.
   *   *التقييم المعماري:* عيب هندسي خطير (`Code Smell`). زيادة في استهلاك الذاكرة، وصعوبة فادحة في الصيانة (إذا أردت تغيير الصوت، ستضطر لتعديله 3 مرات).
2. **الدرجة 2: حلقات التحكم البرمجي (Control Flow Loops - `repeat 3`):**
   *   *التنفيذ:* تغليف كتلة واحدة داخل حلقة `repeat (3)`.
   *   *ماذا يحدث في المعالج؟* يتم تقليل حجم الكود المصصري (`Binary Size`). يقوم المعالج بإعداد مسجل عداد (`Counter Register`) واستخدام تعليمات القفز المشروط (`JMP / JNZ`) لإعادة تشغيل نفس كتلة الذاكرة 3 مرات.
3. **الدرجة 3: التجريد الوظيفي (Procedural Abstraction - `define meow`):**
   *   *التنفيذ:* إنشاء كتلة مخصصة (`Custom Block`) باسم `meow` تحتوي على تشغيل الصوت والانتظار.
   *   *ماذا يحدث في الذاكرة؟* تم عزل المنطق في عنوان ذاكري مستقل (`Isolated Memory Routine`). عند الاستدعاء، يقوم النظام بإنشاء إطار تكديس (`Stack Frame`)، ويدفع عنوان العودة (`Return Pointer`)، وينفذ الوظيفة ثم يعود بسلاسة.
4. **الدرجة 4: التجريد بالمعاملات (Parameterized Functions - `define meow (n) times`):**
   *   *التنفيذ:* تمرير متغير ديناميكي `n` إلى تعريف الوظيفة.
   *   *الذروة الهندسية:* الآن أصبحت الوظيفة عامة وقابلة لإعادة الاستخدام في أي سياق (`Universal Reusable Component`). المتغير `n` يُمرر كمعامل (`Argument`) ويُحفظ في السجلات الذاكرية السريعة للمعالج.

## 📦 الوحدة 12: Advanced Side Effects (`speak` vs `say`)

في التعديل المتقدم، استبدلنا وظيفة العرض النصي `say` بوظيفة النطق الصوتي `speak`.

```text
Function: say()   ──> [ VRAM Pixel Buffer ] ──> Visual Speech Bubble on RGB Monitor
Function: speak() ──> [ OS Speech Engine ]  ──> [ Digital-to-Analog Converter (DAC) ] ──> Sound Waves
```

### ⚙️ الفوارق التحتية للآثار الجانبية (Under the Hood Side Effects):
*   **وظيفة `say(join(hello, answer))`:**
    أثرها الجانبي (`Side Effect`) بصري بحت. يقوم النظام بحساب مصفوفة البيكسلات في الـ `VRAM` ليرسم فقاعة نصية تحتوي على الحروف المشفرة بـ `UTF-8`.
*   **وظيفة `speak(join(hello, answer))`:**
    أثرها الجانبي يتطلب تفاعلاً مع نظام التشغيل (`OS Kernel`). يقوم محرك `Scratch` بتمرير السلسلة النصية إلى محرك تحويل النص إلى كلام (`Text-to-Speech Synthesizer`). يقوم المحرك بتوليد موجات صوتية رقمية (`PCM Audio Waveform`)، ويقوم محول الإشارة الرقمية إلى تناظرية (`DAC`) في بطاقة الصوت بتحويل الأصفار والوحدات إلى ذبذبات كهربائية تحرك غشاء السماعة لينطق اسم "David"!

## 📦 الوحدة 13: Sensor Mechanics: Polling vs Event Interrupts

قدمت المحاضرة طريقتين مختلفتين للتعامل مع الشروط والمستشعرات (`Conditionals & Sensors`). في هندسة النظم الحقيقية، يمثل هذا الفارق ثورة في كفاءة استهلاك الطاقة والمعالج.

```text
Active Polling:  [ Clock Tick ] ──> [ Check Mouse Touch ] ──> [ Clock Tick ] ──> [ Check Mouse Touch ] (High CPU Load)
Hardware Interrupt: [ Video Stream Stream ] ──> (Motion > 10 Threshold Breached) ──> [ Trigger Action ] (Ultra Efficient)
```

### 1. الاستعلام المستمر (Active Polling - `forever if`):
*   *التنفيذ:* استخدام كتلة `forever` بداخلها شرط `if touching mouse-pointer?`.
*   *التحليل الداخلي:* هذه حلقة استعلام نشطة (`Busy Loop / Active Polling`). في كل دورة معالج (عشرات المرات في الثانية)، يقوم النظام بالتحقق من إحداثيات الفأرة مقارنة بإحداثيات الكائن.
*   *العيب الهندسي:* تستهلك طاقة مستمرة من المعالج (`High CPU Utilization`) وتؤدي إلى استنزاف البطارية في الأجهزة المحمولة.

### 2. المقاطعات مدفوعة بالأحداث (Event-Driven Interrupts - `when video motion > 10`):
*   *التنفيذ:* استخدام مستشعر الفيديو العتادي `when video motion > 10`.
*   *التحليل الداخلي:* هنا لا يقوم المعالج بالدوران المستمر في حلقة عمياء. بدلاً من ذلك، يتصل النظام بتيار كاميرا الويب (`Webcam Hardware Stream`). يقوم معالج الصور في الكاميرا بحساب الفروقات البصرية بين الإطارات (`Optical Flow / Motion Frame Delta`).
*   *العبقرية الهندسية:* عندما يتجاوز التغيير الحركي العتبة المحددة (`Threshold > 10`)، يتم إرسال إشارة مقاطعة عتادية (`Hardware Interrupt`) إلى المعالج لتشغيل الصوت `Meow` فوراً. هذه المنهجية هي نفس الأساس المعماري الذي تعتمد عليه الهواتف الذكية عند تشغيل الشاشة بمجرد التقاط الحركة (`Motion Co-processors`)!
