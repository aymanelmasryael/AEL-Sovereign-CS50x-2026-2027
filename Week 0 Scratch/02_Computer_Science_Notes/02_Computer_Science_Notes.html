📦 الوحدة 1: What is Computer Science?

💡 المفهوم الجوهري

علم الحاسب (Computer Science) ليس علم البرمجة، وليس علم صيانة الأجهزة. إنه علم دراسة المعلومات (Information Study):

  • التمثيل (Representation): كيف نترجم الأفكار والأرقام والنصوص والصور إلى تنسيق رقمي قياسي (Standardized Data).
  • المعالجة (Processing): كيف نتلاعب بتلك المعلومات وننقلها من حالة إلى أخرى بأسرع وأكفأ الطرق الممكنة.

📦 الوحدة 2: Computational Thinking (التفكير الحسابي)

💡 تعريفه وأركانه

التفكير الحسابي هو صياغة المشكلات وحلولها بطريقة تمكن الحاسوب (أو الإنسان) من تنفيذها كخطوات منطقية غير قابلة للتأويل.

الأركان الأربعة للتفكير الحسابي:

  • التفكيك (Decomposition): تقسيم المشكلات المعمارية المعقدة إلى مشكلات جزئية يسهل إدارتها.
  • تمييز الأنماط (Pattern Recognition): البحث عن القواعد المتكررة لإعادة استخدام الحلول الكفؤة.
  • التجريد (Abstraction): إخفاء التعقيدات التحتية (مثل كيفية تحريك الإلكترونات في السيليكون) والتركيز على الواجهة الأساسية (مثل الدوال البرمجية).
  • تصميم الخوارزميات (Algorithm Design): بناء خطوات متسلسلة حتمية (Deterministic) تقود حتماً إلى المخرج المطلوب.

📦 Unit 1: What is Computer Science?

💡 The Core Definition

Computer Science is not the study of programming syntax, nor is it computer repair. It is the study of information and computation:

  • Representation: How we translate ideas, numbers, text, images, and audio into standardized digital formats (Data).
  • Processing: How we manipulate this data and transition it from one state to another with maximum efficiency.

📦 Unit 2: Computational Thinking

💡 Definition and Pillars

Computational thinking is formulating problems and their solutions in a way that allows a computer (or human) to execute them as definitive, non-ambiguous logic steps.

The Four Pillars of Computational Thinking:

  • Decomposition: Breaking down complex architectural systems into manageable micro-problems.
  • Pattern Recognition: Identifying recurring rules and data structures to reuse proven, optimized solutions.
  • Abstraction: Hiding underlying execution complexity (e.g., electron physics in silicon) to focus on a clean interface (e.g., function APIs).
  • Algorithm Design: Building deterministic, sequential step-by-step routines that reliably achieve the desired output.

📦 Unit 12: Advanced Side Effects (speak vs say)

In advanced Scratch modifications, we swapped the visual text function say with the audio synthesis function speak.

Function: say()   ──> [ VRAM Pixel Buffer ] ──> Visual Speech Bubble on RGB Monitor
Function: speak() ──> [ OS Speech Engine ]  ──> [ Digital-to-Analog Converter (DAC) ] ──> Sound Waves

⚙️ Under-the-Hood Side Effect Mechanics

say produces a purely visual side effect in VRAM. speak requires OS kernel orchestration to pass the payload to an OS-level Text-to-Speech (TTS) synthesizer, where a hardware DAC translates binary bits into analog voltages that physically vibrate speaker membranes to speak the name "David"!

📦 Unit 13: Sensor Mechanics: Polling vs Event Interrupts

Active Polling:  [ Clock Tick ] ──> [ Check Mouse Touch ] ──> [ Clock Tick ] (High CPU Load)
Hardware Interrupt: [ Video Stream ] ──> (Motion > 10 Threshold) ──> [ Trigger Action ] (Ultra Efficient)
  • Active Polling (forever if): An active busy loop that induces high continuous CPU utilization and thermal throttling.
  • Event-Driven Interrupts (video motion > 10): The CPU sleeps until the webcam image processor dispatches a hardware interrupt upon breaching the motion delta threshold. Peak architectural efficiency!
