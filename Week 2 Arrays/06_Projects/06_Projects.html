مواصفات مشاريع الأسبوع الثاني: سكرابل وقراءة النصوص

المحور الأول: مواصفات وتحديات مشروع سكرابل (Scrabble Specification)

مطلوب بناء برنامج تفاعلي بالطرفية يحسب مجموع النقاط لكلمتين يدخلهما مستخدمان منفصلان، ويعلن فوز أحدهما أو التعادل. 

===================================================================================
             SCRABBLE EXECUTION FLOWCHART
===================================================================================

  [ Input P1 ] ──> compute_score() ──┐
                                     ├──> Comparison ──> [ Print: Player X wins! / Tie! ]
  [ Input P2 ] ──> compute_score() ──┘

===================================================================================
  • معمارية الدالة compute_score: يجب أن تتلقى الدالة السلسلة النصية كمعامل string word وترجع قيمة صحيحة int تمثل مجموع النقاط، مع الالتزام التام بتجاهل أي رموز أو أرقام لا تنتمي للأحرف الأبجدية.

المحور الثاني: مواصفات وتحديات مشروع قراءة النصوص (Readability)

البرنامج يأخذ نصاً كاملاً من المستخدم ويطبق خوارزمية Coleman-Liau الصارمة لتحديد الصف الدراسي المناسب.

  • التقريب الرياضي: استخدام دالة round() من مكتبة <math.h> للحصول على عدد صحيح.
  • إدارة قسمة الأعداد الصحيحة (Integer Division Hazard): عند حساب المتوسطات، يجب تحويل المتغيرات صراحةً إلى float لتجنب فقدان الكسور العشرية.

المحور الثالث: مواصفات مشاريع التشفير (Caesar & Substitution)

شروط مسار التنفيذ لمشروعي قيصر والاستبدال تتضمن تأمين المدخلات والتعامل الجنائي مع مفاتيح التشفير.

  • شروط مشروع قيصر: التأكد من توفر معامل واحد فقط argc == 2، وأن يكون مفتاحاً رقمياً صالحاً. وفي حال إدخال معامل غير صالح، يطبع البرنامج Usage: ./caesar key ويرجع القيمة 1.
  • شروط مشروع الاستبدال: التأكد من أن المفتاح مكون من 26 حرفاً أبجدياً فريداً، مع الحفاظ الكامل على حالة الحروف ومسافات النص الأصلي.

Week 2 Projects Specifications: Scrabble & Ciphers

Section 1: Scrabble Project Specifications

Develop an interactive terminal application that prompts two distinct players for vocabulary strings, computes their cumulative point totals, and displays the definitive victor or a tie state. 

===================================================================================
             SCRABBLE EXECUTION FLOWCHART
===================================================================================

  [ Input P1 ] ──> compute_score() ──┐
                                     ├──> Comparison ──> [ Print: Player X wins! / Tie! ]
  [ Input P2 ] ──> compute_score() ──┘

===================================================================================
  • Architecture of compute_score: The subroutine must accept a character string parameter string word and return a pure integer score, strictly ignoring all numerical digits, punctuation, and whitespace symbols.

Section 2: Readability Project Specifications

The application prompts the user for an expansive text string and applies the strict Coleman-Liau formulation.

  • Mathematical Rounding: Strict utilization of round() from <math.h> to evaluate a rounded integer index.
  • Integer Division Hazard: When calculating averages, developers must explicitly cast integer operands to floating-point values to prevent runtime truncation of decimal precision.

Section 3: Cryptographic Project Specifications

Definitive execution guardrails for the Caesar and Substitution encryption binaries.

  • Caesar Key Verification: The runtime must verify exactly one parameter is passed argc == 2, ensuring it represents a valid numeric key. Otherwise, print Usage: ./caesar key and return exit code 1.
  • Substitution Key Engineering: The runtime must verify the key contains exactly 26 unique alphabetical characters, while preserving cleartext casing and formatting perfectly.
