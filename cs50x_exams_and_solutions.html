Exam Problem 1: The Secret Passcode (Pointers & Stack Pops)

CS50 Midterm / Quiz 0 Difficulty: Hard

Problem Statement: Consider the following snippet of C code. The developer attempts to pass a secret security key between two functions using pointer aliasing. However, the program exhibits unexpected behavior and prints garbage values or crashes with a segmentation fault. Identify the underlying architectural bug and provide the fully corrected C implementation.

// Flawed Examination Code
#include <stdio.h>
#include <stdlib.h>

int* get_secret_code(void) {
    int secret[3] = {42, 101, 50};
    return secret;
}

int main(void) {
    int* code = get_secret_code();
    printf("Secret Code: %d %d %d\n", code[0], code[1], code[2]);
    return 0;
}

AEL Sovereign Expert Solution & Architectural Analysis

Architectural Diagnosis: The bug is a classic Dangling Pointer / Stack Frame Pop. The array secret[3] is allocated locally on the Stack Frame of get_secret_code. When the function returns, its stack frame is popped and invalidated. The returned pointer now references unallocated stack memory, which gets overwritten by subsequent function calls like printf.

Pristine Solution: To persist data beyond a function's scope, the memory must be dynamically allocated on the Heap using malloc, or passed in via caller-owned buffer pointers.

// Corrected Production-Grade Implementation
#include <stdio.h>
#include <stdlib.h>

int* get_secret_code(void) {
    // Dynamically allocate memory on the Heap
    int* secret = (int*)malloc(3 * sizeof(int));
    if (secret == NULL) return NULL;
    
    secret[0] = 42;
    secret[1] = 101;
    secret[2] = 50;
    return secret;
}

int main(void) {
    int* code = get_secret_code();
    if (code == NULL) {
        printf("Fatal Error: Heap exhaustion.\n");
        return 1;
    }
    
    printf("Secret Code: %d %d %d\n", code[0], code[1], code[2]);
    
    // Explicitly free the acquired heap memory to prevent leaks
    free(code);
    return 0;
}

Exam Problem 4: Tri-color Hexadecimal Inversion (Bitmaps & Bitwise Math)

CS50 Final Examination Difficulty: Apex C Master

Problem Statement: In 24-bit BMP images, each pixel is represented by a RGBTRIPLE struct containing rgbtBlue, rgbtGreen, and rgbtRed (each 1 byte). Write a fully optimized C function invert_bitmap_colors that iterates through a 2D array of pixels (height \(H\), width \(W\)) and performs an exact bitwise hexadecimal color inversion on each color channel.

AEL Sovereign Expert Solution & Architectural Analysis

Architectural Analysis: Color inversion in 8-bit unsigned integers (0 to 255) can be achieved mathematically via 255 - value or bitwise via the XOR operator value ^ 0xFF. The bitwise XOR approach compiles down to a single processor instruction, maximizing execution speed across large image buffers.

// Pristine BMP Bitwise Inversion Engine
#include <stdint.h>

typedef struct {
    uint8_t rgbtBlue;
    uint8_t rgbtGreen;
    uint8_t rgbtRed;
} RGBTRIPLE;

void invert_bitmap_colors(int height, int width, RGBTRIPLE image[height][width]) {
    for (int i = 0; i < height; i++) {
        for (int j = 0; j < width; j++) {
            // Bitwise XOR with 0xFF inverts all 8 bits instantly
            image[i][j].rgbtBlue  ^= 0xFF;
            image[i][j].rgbtGreen ^= 0xFF;
            image[i][j].rgbtRed   ^= 0xFF;
        }
    }
}

// Corrected Production-Grade Implementation

#include <string.h>
#include <ctype.h>
#include <strings.h>

#define LENGTH 45
#define N 65536

typedef struct node {
    char word[LENGTH + 1];
    struct node *next;
} node;

extern node *table[N];
unsigned int hash(const char *word);

bool check(const char *word) {
    char lower_word[LENGTH + 1];
    int n = strlen(word);
    
    // Standardize word to lowercase to guarantee identical hash bucket routing
    for (int i = 0; i < n; i++) {
        lower_word[i] = tolower(word[i]);
    }
    lower_word[n] = '\0';

    unsigned int index = hash(lower_word);
    node *cursor = table[index];

        if (strcasecmp(cursor->word, word) == 0) {
            return true;
        }
        cursor = cursor->next;
    }
    return false;
}
