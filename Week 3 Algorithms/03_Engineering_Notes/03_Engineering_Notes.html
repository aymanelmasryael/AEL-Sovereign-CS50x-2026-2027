المحور الأول: الفخ الهندسي لمقارنة السلاسل النصية

عندما تبحث عن رقم في مصفوفة، يمكنك كتابة numbers[i] == target. ولكن إذا كتبت strings[i] == "David" في لغة C، ستحصل على نتيجة كارثية لأن العامل يقارن عناوين الذاكرة بدلاً من الحروف. 

===================================================================================
             STRING EQUALITY MEMORY TRAP
===================================================================================

  if (s == t)  ──>  Compares Address 0x100 with Address 0x200 (Result: FALSE!)
  
  if (strcmp(s, t) == 0) ──> Compares character values 'D','A','V','I','D' (Result: TRUE!)

===================================================================================
  • مكتبة string.h ودالة strcmp: لإجراء مقارنة فعلية للحروف، نستخدم دالة strcmp التي ترجع القيمة 0 إذا تطابقت السلسلتان تماماً.

المحور الثاني: التطبيق العملي لـ typedef struct

لضمان تعرف كافة الدوال على الهيكل المخصص، يتم تعريفه دائماً خارج دالة main في أعلى الملف، واستخدام تدوين النقطة (Dot Notation).

  • تدوين النقطة: نستخدم people[i].name للوصول إلى الحقول الداخلية للكائن.

المحور الثالث: هندسة تحسين الترتيب الفقاعي

  • توفير دورات المعالج: إذا انتهت دورة كاملة دون إجراء أي تبديل، يتوقف البرنامج فوراً، مما يوفر آلاف الدورات المعالجة.

Week 3 Engineering Notes: The String Comparison Trap

Section 1: The String Comparison Trap

When searching for a scalar integer within an array, developers evaluate numbers[i] == target. However, executing strings[i] == "David" in bare-metal C induces a catastrophic logical defect by comparing memory addresses rather than the underlying character stream. 

===================================================================================
             STRING EQUALITY MEMORY TRAP
===================================================================================

  if (s == t)  ──>  Compares Address 0x100 with Address 0x200 (Result: FALSE!)
  
  if (strcmp(s, t) == 0) ──> Compares character values 'D','A','V','I','D' (Result: TRUE!)

===================================================================================
  • strcmp Routine: To evaluate actual character stream equality, developers must invoke strcmp, which returns exactly 0 when both strings match perfectly.

Section 2: Applied typedef struct Implementations

To ensure all function prototypes and subroutines recognize the custom structural declaration, it must be instantiated in the global scope prior to main.

  • Dot Notation: Developers deploy dot notation people[i].name to directly access internal member fields within the struct memory wrapper.

Section 3: Bubble Sort Architectural Optimization

In unoptimized Bubble Sort implementations, the execution loop continues exhaustive comparisons even if the target buffer achieves sorted equilibrium early. To optimize runtime, developers inject a tracking boolean bool swapped.

  • Conserving CPU Cycles: If a complete inner loop executes without a single swap, the execution halts instantly, conserving thousands of processor clock cycles.
